I wanted to get genuinely fluent in AI evaluation, so I built the tool I wished
existed: The Eval Codex.

It's a Claude-tutored skill tree for the AI-eval canon — LLM-as-judge and its
biases, eval datasets and contamination, metrics, agentic/trajectory eval,
benchmarks and Goodhart, statistics for eval, RAG eval, red-teaming. Each topic is
taught through a loop grounded in learning science: you predict your score, get
taught with concrete examples, write it back in your own words, take a cold quiz
graded against a canonical source, and finish on a mock-interview question. Topics
you've mastered resurface on spaced-repetition intervals so the knowledge sticks.

Under the hood it exercised the things an AI-engineering role actually cares about:
the Anthropic API as a stateful multi-step tutor, an API key isolated behind a
localhost-only proxy (never in the browser), learning-science wired in as real
product mechanics, and a full audit trail.

Building the thing you use to learn turns out to be a great way to learn.
